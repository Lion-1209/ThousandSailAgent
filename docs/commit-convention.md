# FireTerminal 提交规范

## 提交信息格式

```
<类型>: <简短描述>

<详细描述>

<关联 Issue>
```

## 类型 (Type)

| 类型 | 说明 | 示例 |
|------|------|------|
| `Initial Commit` | 项目初始化 | Initial Commit: 项目结构搭建 |
| `Add` | 新增功能/文件 | Add: SSH连接管理模块 |
| `Modify` | 修改现有功能 | Modify: 优化终端输出性能 |
| `Fix` | Bug修复 | Fix: 终端中文显示乱码问题 |
| `Refactor` | 代码重构（不改变功能） | Refactor: 抽离存储接口 |
| `Docs` | 文档更新 | Docs: 更新API说明 |
| `Style` | 代码格式化（不改变逻辑） | Style: 统一代码缩进 |
| `Test` | 测试相关 | Test: 添加SSH单元测试 |
| `Build` | 构建系统/依赖更新 | Build: 升级Avalonia到11.3.8 |
| `Revert` | 回退之前的提交 | Revert: 回退上次提交 |

## 简短描述

- 使用中文
- 不超过50个字符
- 不以句号结尾
- 使用祈使句

## 详细描述（可选）

- 说明**为什么**做这个修改
- 列出主要变更点
- 每行不超过72个字符

## 示例

### 简单提交

```
Add: 添加SSH会话管理器
```

### 复杂提交

```
Add: 实现SFTP文件传输

- 添加SftpClient封装
- 支持文件上传/下载进度显示
- 处理大文件传输中断恢复

关联任务: #12
```

### Bug修复

```
Fix: 终端输出编码问题

SSH流返回的UTF-8字符未正确解码，
现在使用Encoding.UTF8显式处理。

修复问题: 终端中文显示乱码
```

## 分支命名

```
feature/<功能名>    # 新功能开发
bugfix/<问题描述>   # Bug修复
hotfix/<紧急问题>   # 生产环境紧急修复
refactor/<模块名>   # 代码重构
docs/<文档类型>     # 文档更新
```

## 提交频率

- 每完成一个逻辑单元即提交
- 提交前确保代码可编译
- 避免提交包含调试代码或TODO

## 禁止事项

- ❌ 提交包含敏感信息（密码、密钥）
- ❌ 提交二进制文件（除必要的资源）
- ❌ 提交自动生成的文件
- ❌ 使用含糊的提交信息（如"修复"、"更新"）
